# Coach Workspace UI Exploration

- Date: 2026-04-13
- Figma file: `Smart Training Workbench - Coach Workspace Exploration`
- Figma URL: `https://www.figma.com/design/v2OIqqdvMmgdDA3bWegbQY`
- Focus: first-pass UI direction for the coach desktop workspace and student H5 recovery check-in

## What was designed

### 1. Coach desktop workspace
- Evidence-first hero with the student name, current goal, active plan version, and primary risk visible immediately.
- Four operational metric cards instead of a KPI-heavy dashboard wall:
  - latest adherence
  - feedback status
  - open alerts
  - draft recovery
- Main content split into one dominant evidence rail and one supporting decision rail:
  - latest execution
  - timeline
  - latest feedback
  - open alerts
  - draft recovery

### 2. Student mobile H5
- A narrow recovery check-in surface focused on signal collection, not engagement theater.
- Explicit inputs for sleep, soreness, and pain, plus a short note field and a single submit action.

## Design choices locked in this exploration
- Warm parchment canvas instead of clinical white.
- Dark ink plus evergreen action color, with copper/rust warning tones.
- Calm, notebook-like cards rather than generic dashboard widgets.
- Desktop-first composition for coaches, mobile-thin composition for students.

## What this exploration resolved
- The product can carry a distinct visual language without becoming decorative.
- The coach workspace should prioritize latest execution and timeline over top-heavy analytics.
- Student H5 can stay intentionally small while still feeling coherent with the main product.

## What still needs iteration
- Some copy and labels should be localized more consistently into Chinese.
- The right rail could become denser or more compact after real data arrives.
- We should decide whether the next screen to design is:
  - the coach dashboard
  - the session logging flow
  - the plan authoring/version diff screen
