# Smart Training Workbench MVP Test Plan

- Plan source: `docs/plans/2026-04-13-smart-training-workbench-mvp-plan.md`
- Branch: `main`
- Base branch: `main`
- Generated by: `/autoplan` review pass

## Affected Pages And Routes

- Coach login
- Coach dashboard
- Student list
- Student workspace
- Assessment form
- Goal form
- Program planner and version history
- Session logging flow
- Student feedback form
- Weekly review page
- Alert/task center
- Admin export and audit views

## Key Interactions To Verify

1. Create student, assessment, goal, and first program version.
2. Recover a session draft after interruption.
3. Log actual execution against a planned session and verify diff generation.
4. Submit student feedback and verify anomaly detection.
5. Open weekly review and verify evidence ordering and source timestamps.
6. Publish a new plan version from a review decision.
7. Resolve an alert and verify audit trail.

## Edge Cases

- Cross-org access attempt.
- Two coaches updating the same student in one review window.
- Feedback submitted after the weekly review was already generated.
- Plan version superseded while an older session is still being logged.
- Attachment upload succeeds but metadata persistence fails.
- Review summary job fails while raw evidence exists.

## Critical Paths

### Path A: New student to first plan
- Auth success
- Student create
- Assessment save
- Goal save
- Program version publish
- Timeline events emitted

### Path B: Planned session to logged execution
- Load active session plan
- Draft execution created
- Planned vs actual diff computed
- Timeline update
- Rule evaluation

### Path C: Feedback to review
- Student feedback submission
- Anomaly detection
- Read model refresh
- Review page render
- Summary provenance links present

## Automated Coverage Targets

- Permission and org-boundary integration tests
- Domain tests for versioning and lifecycle transitions
- Contract tests for timeline events and derived features
- Integration tests for alert rules and suppression
- End-to-end tests for the three critical paths above

## Manual Validation Targets

- Median session logging time for a normal coach workflow
- Coach comprehension of the student workspace top rail
- Student feedback completion rate on phone viewport
- Weekly review usefulness in a real pilot workflow
